# 실험 로그

## 27. PDF/HWP 표·이미지 기반 추가 추출 필요성 점검

### 목적

제공된 RFP 문서에서 표와 이미지를 수치로 확인해, 본문 텍스트 추출만으로 부족할 가능성이 높은 문서를 찾고 OCR 또는 표 추출 후속 실험의 우선순위를 정한다.

### 검증 방법

- 대상: `/data/original_data/files`의 PDF 4개와 HWP 96개, 총 100개 문서
- PDF:
  - `pypdf`로 페이지별 본문 텍스트와 텍스트 미추출 페이지를 확인
  - `PyMuPDF`로 이미지 객체 수와 표 후보 수를 확인
- HWP:
  - HWP 본문 스트림의 표 레코드 수를 확인
  - `BinData`에 포함된 이미지 파일 수를 확인
- 재현 위치: `notebook/01_parser_chunker_test.ipynb`의 27번 섹션

### 결론

- PDF는 텍스트 미추출 페이지가 없어 스캔본 PDF 대상으로 OCR을 일괄 적용할 근거는 확인되지 않았다.
- 다만 전체 100개 문서에서 표 또는 이미지가 확인됐다. 따라서 현재 본문 텍스트 추출만으로는 표 구조와 이미지 안 텍스트를 모두 활용한다고 볼 수 없다.
- 이미지 수가 많다고 즉시 OCR을 적용하지 않는다. 로고·장식 이미지도 포함될 수 있으므로, 이미지 수가 많은 대표 문서에서 이미지 안에 질의에 필요한 글자가 있는지 먼저 확인한다.
- 표가 많은 문서는 OCR보다 표 추출 또는 레이아웃 기반 파싱을 우선 검토한다.

### 한계와 후속 검토

- PDF 표 수는 PyMuPDF가 탐지한 표 후보 수이며, 실제 표 수와 다를 수 있다.
- HWP 표 수는 본문 스트림의 표 레코드 수이며, 표 셀 텍스트가 현재 본문 추출 결과에 모두 포함되는지는 별도 비교가 필요하다.
- HWP `BinData` 이미지 수에는 로고·도형 등 OCR이 불필요한 이미지가 포함될 수 있다.
- 다음 단계에서는 표·이미지가 많은 대표 PDF/HWP를 선정해 원본 화면과 추출 텍스트를 비교하고, OCR 또는 표 추출 도입 여부를 결정한다.
